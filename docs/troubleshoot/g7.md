# G7, ONE+ troubleshooting

See [how to set up G7/ONE+](../../install/g7) first if you haven't already. Many of the underlying causes below are the same as for [G6/ONE](../g6) — signal strength, battery, and OB1 debug logs are covered there and linked from here rather than repeated.

## Step-by-step troubleshooting

If xDrip is set to collect directly from a G7/ONE+ but isn't receiving data, work through these in order:

1. Disable [engineering mode](../../use/misc/#engineering-mode).
2. Uninstall the Dexcom app, or any other app connecting directly to the sensor.
3. Check the sensor itself — can you see the filament protruding from the top? If so, photograph it for Dexcom: even once connected, the sensor won't give useful readings since the filament was never actually inserted under your skin, and you'll need a new one.
4. **"Missing QR Code" on the Dex Status page** — use Settings → Auto Config to scan the QR code from the [setup guide](../../install/g7/#prerequisites), not any code printed on the sensor itself, then tap **Restart Collector** on the Classic Status page.
5. **"Mismatch - wait" on the Dex Status page** — you likely entered the pairing code wrong, or have old used sensors nearby. Move any previously used sensors more than 10 metres away, double-check the pairing code against the one printed on the applicator, then tap **Restart Collector**.
6. **Verify connectivity** — if there's none, use a [Bluetooth scanner](../g6/#how-strong-is-the-transmitter-signal) to check whether the transmitter is visible at all, after moving old devices away. If it's still not visible, hold the applicator's magnet near the sensor for a few seconds — this can trigger it to activate — then check the scanner again and tap **Restart Collector**.
7. **"Deep Sleeping Errors", "Checking Auth Errors", "Scanning Errors" or "Initializing Errors" on the Dex Status page**:
    1. Note down the current 4-digit transmitter ID from Settings.
    2. Change its last digit to something else.
    3. Disable collection in xDrip, and temporarily disable any pump, receiver, or watch connecting directly to the sensor.
    4. Wait 15 minutes for G7 to enter [rapid reconnect mode](#g7-rapid-reconnect-mode).
    5. Delete any Bluetooth devices named `DXCM*`, `DX01*` or `DX02*` from Android's connected/previously-connected device list.
    6. Re-enable the xDrip Dex collector (select Dex as the hardware data source) and restore the correct transmitter ID from your note.
    7. Go to the Dex Status page, wait up to a minute for a pairing request, and approve it — if none appears, restart the phone.
    8. Once connected with 3 consecutive readings, you can re-enable your pump, receiver, or watch.
8. **"Sensor Failed 7" on the Dex Status page** — avoid lying on the sensor and relax the limb it's on, stay hydrated, and give it up to 3 hours for readings to return. If readings do come but keep randomly dropping, see [Intermittent connectivity](../g6/#intermittent-connectivity).

## G7 rapid reconnect mode

Under normal conditions a G7/ONE+ transmits once every 5 minutes, with a two-way acknowledgment handshake each time. If it misses three acknowledgments in a row (15 minutes with no successful handshake), it switches to rapid reconnect mode: it repeats each reading up to 4 extra times, once per minute, so a reconnecting app doesn't have to wait a full 5-minute cycle to catch it. This continues for up to 12 hours, or until the app reconnects — whichever comes first.

As soon as a handshake succeeds, the extra transmissions stop. xDrip reads the timestamp on the reading it gets (which still corresponds to the underlying 5-minute schedule), so it re-synchronises correctly to the normal cadence even if it happened to catch one of the extra once-a-minute transmissions rather than the "real" one.
