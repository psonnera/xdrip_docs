    G7, ONE+

    G7 or ONE+⌁

    This describes how to set up a Dexcom G7 or ONE+ sensor with xDrip. From xDrip's perspective the two are the same device — the setup, grace-period, and troubleshooting steps below apply equally to both.

    Reference documentation is here.

    For G6 or Dexcom ONE, see the separate G6, Dexcom ONE page instead — G7/ONE+ works quite differently since the sensor and transmitter are a single disposable unit.

    Note

    Some of this also applies to Stelo when connected directly to xDrip — check the data source guide for current Stelo support.

    G7 versus ONE+⌁

    There's no functional difference for xDrip: same connection method, same steps below. The differences between G7 and ONE+ (accessories, app features, pricing) are Dexcom's own product-line differences and don't change how you set them up here.

    Also see Dexcom basics on the G6/ONE page — phone clock sync, battery optimization, and not pairing from Android Bluetooth settings all apply here too.

    t:slim / receiver order matters for G7

    Connect with xDrip first. If you connect a pump or receiver to the sensor before xDrip, xDrip will not be able to connect to it.

    Prerequisites⌁

    • Make sure xDrip is updated to at least the 2026.05.06 release or newer.
    • Use Settings → Auto configure to scan the G7/ONE+ QR code shown in the recommended settings guide. This is a one-time setup — it doesn't need to be repeated for each new sensor — but review the imported settings yourself, since scanning the QR code doesn't confirm them for you. Approve the prompt it shows.
    • Go to Android Settings → Connected devices and forget any devices named DXCM**, DX01** or DX02** (where ** is any two characters) before starting a new sensor.

    Starting (inserting) the sensor⌁

    Before inserting a new sensor, mark the old one with a pen so you don't accidentally remove the wrong one later. Inspect the new sensor for damage, then insert it following the manufacturer's applicator instructions.

    The session starts automatically as soon as the sensor is inserted — there's no need to manually start or stop anything in xDrip for this step. Take note of the 4-digit pairing code shown on the sensor applicator; you'll need it below.

    Establishing connectivity⌁

    1. Only one app can collect from the transmitter at a time. If you have the Dexcom app, or any other app that collects directly, uninstall it. Don't start a Dexcom receiver, or a pump that reads G7 directly, yet — if either is already running, disable it and wait 15 minutes.
    2. Go to Android Settings → Connected devices and forget any devices named DXCM**, DX01** or DX02**.
    3. Confirm xDrip is at least the 2026.05.06 release.
    4. Complete the Prerequisites settings above — review them yourself, the QR code doesn't configure everything automatically.
    5. Make sure old sensors are out of range (10+ metres) or shielded (a microwave works). A still-active old sensor is fine as long as it stays connected to a different xDrip instance or the Dexcom app; otherwise peel it off and move it away.
    6. Enter the 4-digit pairing code from the sensor applicator as the transmitter ID.
    7. Go to the Dex Status page and wait for a connection. On Android 10+, approve the pairing request when it appears.

    If the sensor was inserted more than 30 minutes ago you should see readings right away; if it was just inserted, it will show as "warming up" and the status page will fill in over time (remaining time, etc.). Once the Dex Status page shows battery voltages, it's safe to re-enable your pump or receiver.

    Grace period⌁

    A G7/ONE+ device runs for 10.5 days — the 12 hours beyond the normal 10-day session is what Dexcom calls the grace period. xDrip shows a sensor as expiring after 10.5 days. You can use this extra time in a few ways, and combine them:

    • Flexibility — use each sensor the full 10 days as normal, and treat the extra 12 hours as a buffer for the day you can't start a new sensor exactly on time.
    • Extended run time — deliberately run each sensor the full 10.5 days: if you start one at 7:00 AM, start the next at 7:00 PM, alternating morning/evening starts. Over 9 sensors that adds up to 94.5 days instead of 90.
    • Presoak — insert the new sensor while the old one is still active (e.g. insert at 7:00 AM, let the old one keep running until 7:00 PM), so the new sensor has already been in for a few hours — and past its least stable initial readings — by the time it takes over.

    Starting a subsequent sensor⌁

    If this is your first G7/ONE+ sensor, use Establishing connectivity above instead.

    When the status page shows 10 transmitter days, or the optional sensor-expiry note shows about 12 hours left, insert the new sensor (see Starting the sensor above) — do this around the 10-day mark rather than waiting for the full 10.5-day grace period to end, so the two sensors overlap smoothly.

    You can optionally verify the new sensor is working before switching over: use a secondary phone, or a second xDrip variant on the same phone, to run Establishing connectivity against the new sensor (if using a variant on the same phone as your active main app, don't forget the Bluetooth devices from the list). Once confirmed, disable that secondary connection again.

    When the old sensor actually stops (at 10.5 days or earlier):

    1. If you used a secondary phone/variant to verify the new sensor, disable its collection now and wait 15 minutes for G7 rapid reconnect mode to kick in.
    2. Carefully peel off the old sensor without disturbing the new one. Store it in a metal box, or move it at least 10 metres away, to avoid interference.
    3. On your main phone, follow Establishing connectivity again using the new sensor's pairing code.

    Stopping a sensor⌁

    There's no manual "Stop Sensor" option for G7/ONE+, unlike G6. A session starts automatically the moment you insert the sensor, so there's nothing to manually start — and since the sensor and transmitter are a single disposable unit, once a session ends it can never be restarted, so a manual stop was removed from xDrip to prevent accidentally disabling a functional sensor for nothing.

    Stopping the session (if it were possible) wouldn't turn off the transmitter's Bluetooth anyway — like G6, it keeps transmitting every 5 minutes for as long as the battery lasts (100+ days), session or no session.

    If you want to remove a sensor you're done with, disable collection in xDrip before peeling it off. Otherwise xDrip may record one last, very low reading as the sensor is pulled from your skin, which can skew your glucose history.

    Please report unknown firmware⌁

    If the firmware version shown on the System Status page is flagged as unknown, check the release notes first — if a newer xDrip release already added support for it, update to that version. If it's genuinely not listed anywhere, please let the developers know.
